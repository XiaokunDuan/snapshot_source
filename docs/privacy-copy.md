# Privacy and Permission Copy

Use this copy as the source of truth for App Store permission strings and release-facing privacy text.

## Camera

`Snapshot needs camera access so you can capture a scene and turn it into a study card.`

## Photo Library Read

`Snapshot needs photo library access so you can import an image into your study flow.`

## Photo Library Add

`Snapshot may save generated images or exports back to your photo library when you choose to save them.`

## Public Privacy Summary

- Snapshot uses images you choose to capture or import in order to generate study content.
- Snapshot stores your study history so you can revisit it later.
- Snapshot may keep account and subscription state needed to operate the app.
- Any third-party services used for analysis or sign-in should be described in the public privacy policy before submission.

## Review Framing

- Keep permission explanations tied to a user action.
- Avoid vague phrases like "for better experience".
- If a permission is optional, say so plainly in the UI and the review notes.
